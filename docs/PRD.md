# PRD

## 项目名称

美甲预约微信小程序

## 产品目标

为用户提供便捷的美甲服务浏览、预约下单与到店管理体验；为店主提供基础预约信息查看能力。

## 目标用户

- C 端用户：浏览服务、查看热门款式、选择美甲师、提交预约
- 店主 / 店员：查看预约信息、了解热门款式展示内容、确认服务安排

## V1 范围

- 首页展示门店与服务项目
- 首页新增热门款式推荐区
- 服务项目列表
- 预约表单
- 预约页支持选择美甲师
- 预约提交
- 预约列表查看
- 基础联系信息收集

## 本次需求范围定义

### 1. 首页新增热门款式推荐区

目标：提升首页内容丰富度，帮助用户更快被具体款式吸引并进入预约流程。

范围内：
- 在首页新增“热门款式推荐”区块
- 展示 3~6 个推荐款式卡片
- 每个卡片至少包含：款式名称、图片、标签/风格描述、价格参考或关联服务
- 支持点击“立即预约”跳转预约页

范围外：
- 个性化推荐算法
- 按用户画像推荐
- 款式收藏、点赞、分享
- 款式详情页

### 2. 预约页支持选择美甲师

目标：让用户在预约时可表达对服务人员的偏好，提高预约确定性与转化率。

范围内：
- 在预约页新增“美甲师”选择项
- 展示当前可预约的美甲师列表
- 支持选择具体美甲师，或选择“无偏好/到店安排”
- 提交预约时写入所选美甲师信息
- 预约列表可展示已选择的美甲师信息

范围外：
- 美甲师排班系统
- 按美甲师实时过滤可约时间段
- 技师评分、简介详情页
- 多门店技师调度

## 暂不纳入 V1

- 在线支付
- 会员体系
- 积分/优惠券
- 多门店复杂调度
- 实时 IM 客服
- 个性化推荐算法
- 技师排班与产能计算

## 核心用户流程

1. 用户进入首页
2. 查看热门款式推荐与服务项目
3. 点击首页“立即预约”或款式卡片进入预约页
4. 选择服务项目
5. 选择美甲师或“不指定”
6. 选择预约日期与时间段
7. 填写姓名、手机号、备注
8. 提交预约
9. 查看预约结果或已有预约记录

## 成功标准

- 用户可以在 3 分钟内完成一次预约提交
- 首页可正常展示热门款式推荐区，且不影响现有服务项目展示
- 用户可在预约页完成美甲师选择并成功提交预约
- 店主可查看预约数据中的美甲师信息
- 前后端接口与字段定义稳定
- 架构师可以将需求拆成前后端独立可执行任务

## 当前假设

- 当前先以单门店为范围
- 服务项目、热门款式、美甲师列表先用静态配置或轻量后端存储
- 热门款式为人工配置内容，不做个性化推荐
- 预约审核流程先不做复杂状态机，仅保留基础状态字段
- 美甲师选择在 V1 仅表示用户偏好，不引入排班冲突校验
- 预约页默认提供“无偏好/到店安排”选项，避免因列表不可用阻塞预约

## 待确认问题

- 热门款式是否需要后台可配置排序与上下架
- 热门款式卡片是否必须包含图片素材，还是允许先用占位图
- 美甲师选择是否只展示姓名，还是需要展示擅长风格 / 级别
- 是否要求预约列表和店主侧都展示美甲师字段
- 后续是否要升级为“按美甲师可约时间段”模式
