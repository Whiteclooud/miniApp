# WORKFLOW

## 沟通方式

- 你只需要在飞书和架构师对话
- 架构师负责需求澄清、范围控制、任务拆解、风险判断、交付评审
- 前端和后端默认不直接对外沟通

## 标准工作流

1. 你用自然语言描述需求
2. 架构师判断是否需要澄清问题
3. 架构师更新 `PRD / ARCHITECTURE / API / TASKS`
4. 架构师给前端/后端下发任务 brief
5. 前后端修改各自 workspace 中的代码 clone
6. 架构师汇总结果、评审、反馈

## 对架构师的输出要求

每次接到新需求，默认输出以下结构：

- 目标
- 范围内 / 范围外
- 用户流程
- 数据对象
- 接口改动
- 前端任务
- 后端任务
- 风险与待确认问题
- 验收标准

## 对前后端的 handoff 要求

每个任务至少包含：

- 任务 ID
- owner
- 背景
- 要改哪些文件
- 不能改什么
- 完成标准
